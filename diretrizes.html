DIRETRIZES



O projeto Espectro Web foi desenvolvido sendo tomado como base a WCAG (Web Content Accessibility Guidelines), nela temos diretrizes essenciais para garantir que o conteúdo online seja acessível a todos os usuários, independentemente de suas habilidades ou deficiências, dentro da WCAG foi retirada algumas diretrizes direcionadas especificamente ao publico autista. Partindo deste pressuposto, podemos falar que a WCAG é fundamentada em quatro princípios-chave:

    - Perceptível
    - Operável
    - Compreensível
    - Robusto.

Vamos falar especificamente sobre eles direcionando-os ao publico TEA:

1. Perceptível:

Este princípio ressalta a importância do conceito sobre assegurar qua a interface do usuário(UI) e o conteudo do seu produto seja percebido de maneira fácil e cristalina por todos os usuários. Em outras palavras, tornar o conteúdo disponível em formas que podem facilmente compreendida pelo usuário para atender usuários com diversos tipos de dificuldades sensoriais.


Parte das pessoas autistas podem processar as informações de maneira visual, e fornecer texto alternativo para imagens permite que elas entendam melhor o conteúdo, especialmente se tiverem dificuldades com linguagem visual ou simbólica.

Acesse os princípios perceptíveis

2. Operável:

Este princípio parte do pressuposto de fornecer garantia para que os usuários possam operar a interface do usuário e navegar pelo conteúdo de forma eficaz. Isso inclui tornar todos os elementos interativos acessíveis via teclado, para atender às necessidades de usuários com deficiências motoras que não podem usar um mouse. Além disso, é importante fornecer tempo suficiente para que os usuários possam interagir com o conteúdo e evitar elementos que possam causar distração ou desconforto durante a navegação.

Acesse os princípios operáveis

3. Compreensível:

Este princípio trata da clareza e compreensibilidade do conteúdo e da interface do usuário. Isso envolve, exemplo, a utilização de uma linguagem simples e clara.

Acesse os princípios compreensíveis

4. Robusto:

O princípio da robustez diz respeito à garantia de que o conteúdo seja compatível com uma variedade de tecnologias de assistência e dispositivos de usuário.

Acesse os princípios Robustos

>Esses princípios são inter-relacionados e trabalham juntos para fornecer uma experiência digital mais acessível e inclusiva para todos os usuários. Ao seguir esses princípios e as diretrizes associadas, os desenvolvedores podem criar sites e aplicativos que atendam às necessidades de uma ampla variedade de usuários, promovendo assim a inclusão e a equidade na web.
